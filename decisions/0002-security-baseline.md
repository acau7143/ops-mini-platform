\# Decision 0002 — SSH security baseline (phase 1)



\## Context

운영 서버의 기본 원격 접근 방식은 SSH이다.

초기 단계부터 최소한의 SSH 보안 기준을 정리할 필요가 있다.



\## Decision

1차 SSH 보안 베이스라인으로 `PasswordAuthentication no` 를 적용한다.

SSH 접속은 키 기반 인증을 기본으로 사용한다.



\## Why

\- 비밀번호 로그인은 추측 공격 대상이 될 수 있다.

\- 현재 환경은 PEM 키 기반 접속이 이미 가능하다.

\- 작은 변경으로도 기본 보안 수준을 높일 수 있다.

\- 초반에는 과도한 변경보다 검증 가능한 최소 변경이 적합하다.



\## Safety

\- 기존 SSH 세션을 유지한 상태에서 설정을 수정한다.

\- 설정 변경 후 `sudo sshd -t` 로 문법 검사를 수행한다.

\- 검사 성공 후 `sudo systemctl reload ssh` 를 적용한다.

\- 새 세션 재접속 성공을 확인하기 전까지 기존 세션을 종료하지 않는다.

\- 재접속 실패 시 백업 파일로 즉시 복구한다.



\## Result

기본적인 SSH 접근 보안을 강화하면서도 원격 잠금 리스크는 낮게 유지하는 방향으로 결정했다.

# Decision 0001 — 프로젝트 목표 및 범위



## 목표(증명하고 싶은 역량 3개)

- 리눅스 서버 운영 기본기(서비스/로그/권한/네트워크 점검)

- 장애 재현 → 원인 분석 → 복구를 문서로 남기는 능력

- IaC/자동화로 재현 가능한 인프라를 만드는 능력



## 12주 후 성공 기준(눈에 보이는 결과 3개)

- Terraform으로 EC2/보안그룹 등 인프라를 재현 가능(`apply/destroy` 반복)

- Runbook(운영/점검/복구) 작성 + Incident(장애 보고서) 10개 이상

- 모니터링/알람이 실제로 동작하고(증거 포함) 대응 절차가 문서화됨



## 범위에서 제외(이번 프로젝트에서 안 하는 것 2개)

- 웹 서비스 기능 개발(백엔드는 최소 기능만 사용)

- AWS 서비스를 “많이 나열”하는 구성(운영/장애대응이 중심)



## 이유(왜 이 프로젝트인가)

- 시스템엔지니어 업무에서 중요한 “운영/장애대응/재현성”을 포트폴리오로 증명하기 위해

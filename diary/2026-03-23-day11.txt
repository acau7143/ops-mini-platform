## 오늘 한 일

- deploy.md에 Nginx 설정 변경과 Docker 앱 업데이트 절차를 각각 정리했다.
- rollback.md에 Nginx 롤백과 Docker 롤백 절차를 작성했다.
- nginx 설정 변경 후 reload를 생략했을 때 변경이 반영되지 않는 상황을 재현했다.
- nginx -T 명령으로 실제 nginx가 읽고 있는 설정과 파일 내용이 다를 수 있다는 점을 확인했다.
- INC-007을 작성하고 runbook/index.md에 deploy.md, rollback.md를 추가했다.

## 오늘 배운 점

- `nginx -t`는 문법 검사만 한다. 실제 반영은 `systemctl reload nginx`가 해야 한다.
- reload를 안 하면 nginx 프로세스는 이전 설정을 메모리에 그대로 들고 동작한다.
- `nginx -T`를 쓰면 현재 nginx가 실제로 읽고 있는 설정 전체를 출력할 수 있어서,
  파일 내용과 실제 동작 설정이 다른지 비교할 때 유용하다.
- 배포 절차는 변경 → 검사 → 반영 → 검증 4단계가 빠짐없이 있어야 한다.
- 설정 파일에 오류가 있으면 nginx -t도 실패하고 reload도 거부된다.

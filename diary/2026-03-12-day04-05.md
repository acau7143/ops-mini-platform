# Day4-5 — Nginx 설정 검증 절차 + SSH 보안 베이스라인 1차



## 목표

Nginx 설정 변경 전에 백업과 문법 검사를 수행하는 절차를 고정하고

SSH 접근 보안 설정을 1차로 적용하면서도 서버 접속이 끊기지 않도록 안전 적용 절차를 정리한다.



## 오늘 한 일

- `/etc/nginx/sites-available/default` 파일 백업 생성

- `sudo nginx -t`로 현재 nginx 설정 정상 여부 확인

- nginx 설정 파일에 의도적인 문법 오류 1줄을 넣어 검사 실패를 재현

- 백업 파일로 원복 후 다시 `sudo nginx -t` 성공 확인

- `runbook/access.md`에 nginx 설정 변경 전 백업/검증 절차 추가

- SSH 설정 파일(`/etc/ssh/sshd_config`) 백업 생성

- SSH 설정 1차 하드닝(시스템을 더 안전하게 만드는 설정 강화)으로 `PasswordAuthentication no` 적용

- `sudo sshd -t`로 SSH 설정 문법 검사 수행

- `sudo systemctl reload ssh` 후 새 세션에서 SSH 재접속 성공 확인

- `runbook/access.md`에 SSH 설정 변경 시 안전 적용 절차 추가



## 오늘 배운 점

- 서비스 설정은 수정 자체보다 적용 전 검증 절차가 더 중요하다.

- nginx는 `nginx -t`로 설정 오류를 reload 전에 차단할 수 있다.

- SSH 설정은 잘못 바꾸면 원격 접속이 끊길 수 있으므로 기존 세션 유지가 필수다.

- 보안 설정은 한 번에 많이 바꾸기보다 작은 변경부터 안전하게 검증하는 것이 좋다.

- 운영에서는 “변경 → 검사 → 적용 → 재검증” 흐름이 중요하다.




## 결과/증거

- `sudo nginx -t` 실패 출력 캡처

- `sudo nginx -t` 성공 출력 캡처

- 새 SSH 세션 재접속 성공 기록 또는 스크린샷


## 막힌 점

- 단순히 설정값만 아는 것보다 기존 세션을 유지하면서 새 세션으로 검증하는 방식이 더 중요하다는 점을 확인했다.

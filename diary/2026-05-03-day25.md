## 왜 이 기술/방법을 선택했는가
 
| 항목 | 선택한 것 | 대안 | 선택 이유 |
|------|-----------|------|-----------|
| IaC 도구 | Terraform | AWS CloudFormation | AWS 외 GCP, Azure도 쓸 수 있어서 범용성이 높다 |
 
## 오늘 처음 안 사실
 
| 몰랐던 것 | 알게 된 것 | 왜 그렇게 되는지 |
|-----------|-----------|----------------|
| IaC랑 Terraform이 같은 건 줄 알았다 | IaC는 개념이고 Terraform은 그 개념을 구현한 도구다 | Docker Compose도 IaC의 일종인 것처럼, 도구와 개념은 다르다 |
| `terraform plan`이 설정 파일인 줄 알았다 | plan은 실제 생성 전에 뭐가 만들어질지 미리 보여주는 명령어다 | AWS에 아무것도 안 만들고 미리보기만 해줘서 실수를 방지한다 |
| `~> 5.0`이 딱 5.0 버전만 쓰는 줄 알았다 | 5.x 범위 안에서 최신 버전을 자동으로 설치한다 | 매번 버전 바꿀 필요 없이 호환 가능한 최신 버전을 쓸 수 있다 |
| `provider.tf`가 뭔지 몰랐다 | Terraform한테 "나 AWS 쓸 거야, 서울 리전이야"를 알려주는 파일이다 | Terraform은 AWS 외에도 GCP, Azure를 쓸 수 있어서 어떤 클라우드인지 명시해야 한다 |
| `ingress`, `egress`가 뭔지 몰랐다 | ingress는 들어오는 트래픽, egress는 나가는 트래픽이다 | in/out 방향으로 트래픽을 구분해서 허용/차단을 설정한다 |

## 막힌 점
- terraform plan이 명령어인지 설정 파일인지 처음에 헷갈렸다.
- provider.tf가 뭘 하는 파일인지 몰랐는데,
  Terraform이 어떤 클라우드를 쓸지 알려주는 파일이라는 걸 이해했다.

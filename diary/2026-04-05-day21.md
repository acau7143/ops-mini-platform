## 이번 주 배운 점

- 보안 설정은 "적용했다"만으로 끝나지 않는다. 탐지 → 재현 → 복구 → 자동화까지 이어져야 의미가 있다.
- `access.log`에서 외부 스캐너 IP를 직접 발견했다. 실습 환경이라도 실제 스캔은 존재한다.
- nginx `add_header`는 `location` 블록에 자체 헤더가 있으면 상위 블록 헤더가 무시된다.
- `chmod 777`은 느슨하게 준 것이 아니라 누구나 수정 가능한 상태다. 운영 서버에서는 실제 위협이 된다.
- SLA 기준은 시간 숫자보다 행동 기준("즉시 확인 / 당일 해결")이 솔로 실습 환경에서 더 현실적이다.
- `inactive` 안에 `active`가 포함되어 있어서 grep 조건을 구체적으로 써야 한다는 점을 직접 버그로 확인했다.
- Docker Compose 멀티 컨테이너에서는 서비스 이름이 곧 호스트명이 된다.

## 아쉬운 점

- 보안 헤더 적용 후 브라우저 개발자 도구로 실제 응답 헤더를 확인하는 걸 놓쳤다.
- 스크립트 결과를 JSON이나 표 형태로 저장하면 주차별 비교가 가능할 것 같다.
